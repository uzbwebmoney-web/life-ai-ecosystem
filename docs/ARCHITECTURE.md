## Слои

| Слой | Назначение |
|------|------------|
| **Gateway** | Telegram, голос, фото, inline-меню |
| **Module Router** | Определяет модуль по намерению или команде |
| **Modules** | Доменная логика (health, finance, car…) |
| **Core** | Общие сервисы для всех модулей |
| **Data** | Профили, записи, файлы, embeddings |

## Cross-cutting (работает во всей экосистеме)

| Возможность | Сервис |
|-------------|--------|
| Персональная память AI | `services/memory` |
| Поиск по жизни | Memory + structured records |
| Голос | Gateway → STT/TTS |
| OCR / фото | `services/documents` |
| Семейные профили | `models` + family scope |
| Единая панель | Dashboard module |
| Напоминания | `services/notifications` |

## Модуль (контракт)

Каждый модуль реализует:

- `id` — стабильный ключ (`health`, `finance`…)
- `title` — название для меню
- `router` — aiogram Router с handlers
- `menu()` — кнопки подмодуля
- `ai_context()` — системный промпт для домена
- `record_types()` — типы записей для поиска и памяти

## AI-маршрутизация

1. Пользователь пишет свободный текст / голос / фото.
2. **Intent router** (лёгкая модель или rules) выбирает модуль.
3. Модуль + core memory формируют ответ.
4. При необходимости — сохранение записи (с согласия).

## Без внешних интеграций

Экосистема не зависит от API банков, Google Calendar, почтовых сервисов и облаков пользователя.

| Данные | Как попадают в систему |
|--------|-------------------------|
| События календаря | Пользователь добавляет в боте |
| Кредиты и платежи | Ручной ввод суммы, платежа/мес, числа месяца |
| Напоминания | `/remind` + фоновый worker |
| Чеки и документы | Фото/файл в Telegram |
| Погода | Open-Meteo (без ключей) |

## Безопасность

- Медицина и юриспруденция — disclaimers, не замена врача/юриста.
- PII шифруется at rest.
- Память AI — opt-in на каждый тип данных.
- Экстренные сценарии — только справочная информация + номера служб.
